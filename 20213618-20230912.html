git bash实验报告

一、 git、shell、git bash简介
git
版本控制工具，作用是将本地代码备份到服务器，程序员可以在任何有电脑和网络的地方将代码下载下来完成编程工作。主要功能有合作开发、上传代码到服务器、下载代码到本地、创建与合并分支等其本质是一个命令行的命令，跟ls、mv、rm等命令是一样的。
shell
命令行，谈到命令行时，我们实际上指的是shell。shell是一个程序，并将它们传递给操作系统执行。几乎所有的Linux发行版都提供了一个来自GNU项目的shell程序。
git bash
windows的命令提示符(cmd)不能直接使用git命令，git bash就是在windows下使用git命令的模拟环境终端，windows通过git bash模拟命令行在unix系统下的执行环境，通过这个模拟环境来执行git命令。

三、总结
一个好的命令行界面是一种与计算机通信的出色表达方式，就像人类的文字一样。据说，“图形用户界面使简单的任务变得容易，而命令行界面使困难的任务变得成为可能”，“学习Shell”，开始了我们对命令行的基本语言的探索，包括命令的结构、文件系统导航、命令行编辑，以及为命令寻找帮助和文档。
Git Bash是一个命令行工具，它提供了一个类Unix的Shell环境，以下为我学习的一些常见的Linux命令：
cd切换目标 如：进入D目录：cd /d；进入D目录下的子目录lk：cd /D/lk；
ls ( list ) 查看当前目录下的内容
ls -al 包括隐藏文件和以 . 开头的文件
mkdir创建目录
rmdir删除文件夹；或者rm -rf（目录可以有文件）
touch 创建文件：touch lk.txt
cat 查看文件内容 :cat lk.txt
cp ( copy ) 复制文件
cp -r A/* B 复制 A 文件夹下的所有子文件、目录到 B 文件夹
rm ( remove ) 删除文件
mv ( move ) 移动文件或重命名
将文件放入指定文件夹中： mv /d/*.后缀 文件夹名字
等等
